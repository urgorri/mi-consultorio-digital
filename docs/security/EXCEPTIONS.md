# Security Exceptions

This document tracks security vulnerabilities that have been identified but are temporarily accepted.

## Policy

- All high and critical vulnerabilities must be resolved or documented here.
- Exceptions must include a reason and an expiration date.
- Upon expiration, the exception must be re-evaluated and either resolved or extended with a new justification.
